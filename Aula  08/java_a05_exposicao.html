Exceções como
Controle de Fluxo

Exceções não são apenas erros — são um mecanismo de controle de fluxo que redireciona a execução quando algo inesperado ocorre. Domine try/catch/finally, a hierarquia de exceções, throws e como criar suas próprias exceções.

O que é uma Exceção

Exceção = desvio de fluxo em situação anormal

Quando Java encontra uma situação que não pode tratar normalmente — divisão por zero, arquivo não encontrado, índice inválido — ele lança uma exceção: cria um objeto especial, para a execução normal e procura um tratador (catch) na pilha de chamadas.

Por que isso é controle de fluxo? Um try/catch redireciona a execução assim como um if/else — mas para situações excepcionais. Sem try/catch, uma exceção não tratada encerra o programa. Com ele, você decide o que fazer.
try / catch / finally

A estrutura básica de tratamento

O bloco try contém o código que pode falhar. O catch captura a exceção e define o que fazer. O finally sempre executa — com ou sem exceção — ideal para fechar recursos.

JAVA — try / catch / finally
import java.util.Scanner;

public class ExemploBasico {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        try {
            // Bloco try: código que pode lançar exceção
            System.out.print("Digite um número: ");
            int n = sc.nextInt();           // InputMismatchException se não for int
            System.out.println("100 / " + n + " = " + (100 / n));  // ArithmeticException se n=0

        } catch (ArithmeticException e) {
            // Captura SOMENTE ArithmeticException (divisão por zero)
            System.out.println("Erro: divisão por zero! " + e.getMessage());

        } catch (java.util.InputMismatchException e) {
            // Captura SOMENTE InputMismatchException (entrada inválida)
            System.out.println("Erro: digite um número inteiro!");

        } finally {
            // finally SEMPRE executa — com ou sem exceção
            // Ideal para fechar recursos: Scanner, arquivos, conexões
            sc.close();
            System.out.println("Scanner fechado.");
        }
    }
}
💡
Múltiplos catch — ordem importa: o Java testa os catches de cima para baixo e executa o primeiro compatível. Sempre coloque exceções mais específicas antes das mais genéricas — ArithmeticException antes de Exception.
Hierarquia de Exceções

Checked vs Unchecked

Todas as exceções em Java herdam de Throwable. O compilador exige tratamento apenas para Checked exceptions (verificadas) — as que herdam de Exception mas não de RuntimeException.

⚠️
Unchecked não significa ignorável: RuntimeExceptions não obrigam try/catch, mas causam crashes se não tratadas. Use-as para indicar bugs de programação (índice inválido, argumento nulo). Checked exceptions indicam situações externas previsíveis (arquivo ausente, rede indisponível).
throws e throw

Propagando exceções

Em vez de tratar a exceção localmente, um método pode propagá-la para quem o chamou usando throws na assinatura. Para lançar intencionalmente, use throw new.

JAVA — throw e throws
// throws na assinatura: avisa o compilador que este método pode lançar
static double dividir(double a, double b) throws ArithmeticException {

    // throw new: lança intencionalmente uma exceção
    if (b == 0) throw new ArithmeticException("Divisor não pode ser zero");

    return a / b;
}

public static void main(String[] args) {
    try {
        System.out.println(dividir(10, 2));   // 5.0
        System.out.println(dividir(10, 0));   // lança!
    } catch (ArithmeticException e) {
        System.out.println("Capturado: " + e.getMessage());
    }
}
Exceções Customizadas

Criando suas próprias exceções

Quando nenhuma exceção padrão descreve bem o problema do seu domínio, crie a sua. Estenda RuntimeException para unchecked ou Exception para checked.

JAVA — exceção customizada
// Exceção customizada — estende RuntimeException (unchecked)
class SaldoInsuficienteException extends RuntimeException {

    private final double saldoAtual;
    private final double valorSolicitado;

    public SaldoInsuficienteException(double saldoAtual, double valorSolicitado) {
        // super() passa a mensagem para RuntimeException
        super(String.format("Saldo insuficiente: tem R$%.2f, precisa R$%.2f",
            saldoAtual, valorSolicitado));
        this.saldoAtual      = saldoAtual;
        this.valorSolicitado = valorSolicitado;
    }

    // Getters para quem capturar poder acessar os detalhes
    public double getSaldoAtual()      { return saldoAtual; }
    public double getValorSolicitado() { return valorSolicitado; }
}

class ContaBancaria {
    private double saldo;

    public ContaBancaria(double saldo) { this.saldo = saldo; }

    public void sacar(double valor) {
        if (valor > saldo)
            throw new SaldoInsuficienteException(saldo, valor);
        saldo -= valor;
        System.out.printf("Saque de R$%.2f realizado. Saldo: R$%.2f%n", valor, saldo);
    }
}

// Uso:
ContaBancaria conta = new ContaBancaria(500);
try {
    conta.sacar(200);   // OK
    conta.sacar(400);   // lança SaldoInsuficienteException
} catch (SaldoInsuficienteException e) {
    System.out.println(e.getMessage());
    System.out.printf("Você pode sacar até R$%.2f%n", e.getSaldoAtual());
}
